# Ivory Editorial Portfolio Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Preserve the portfolio’s content and interaction contracts while replacing its neon cyber styling with the approved Ivory Editorial light theme and deep-navy dark theme.

**Architecture:** Keep the existing static HTML and JavaScript module boundaries. Add a dependency-free Node regression test that checks stable DOM hooks, theme tokens, forbidden cyber styling, and drag-removal requirements; then refactor CSS and the small effect/theme surfaces until those tests pass. Finish with browser verification at desktop and mobile sizes.

**Tech Stack:** Static HTML5, CSS custom properties, vanilla JavaScript, Node.js built-in test runner, local HTTP server, browser automation.

## Global Constraints

- Keep existing content, links, IDs, `data-*` attributes, tab/filter logic, language switching, theme switching, modal behavior, copy buttons, and contact form behavior.
- Preserve OS-based initial theme selection and saved `localStorage` theme precedence.
- Use `#F7F3EA`, `#FAF8F3`, `#FFFFFF`, `#071A2F`, `#111827`, `#6B7280`, `#B88A44`, and `#C6A15B` as the light-theme foundation.
- Use deep navy and charcoal rather than black for dark mode.
- Remove card drag/reorder behavior and its visual affordances.
- Retain particles, typing, subtle application-card tilt, and detection-style hover only after restyling them as quiet luxury details.
- Do not modify `assets/data.json`, thumbnail files, external destinations, or content copy.
